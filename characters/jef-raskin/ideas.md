## Flagship frame

**Read the intent, not the Mac.** Raskin started the Macintosh and spent twenty years explaining
why the shipped product betrayed the humane interface. *If* Alan Kay or Don ever join a produced
episode, Dynabook-vs-Mac and humane-repo inspectability are natural threads — ask first.

### 1. Locus of attention × Repo Show navigation

On air: land on a **directory listing** (view source) → open **`README.md`** (humane overlay).
Same URL path; different projection. Tie to Raskin's **monotonicity** rule — don't decrease the
user's cognitive position when switching views. Depth: [`humane-interface-and-this-repo.md`](humane-interface-and-this-repo.md)

### 2. *The Humane Interface* — cognetics on air

Walk the book's load-bearing ideas — modelessness, **~7±2**, zoom vs menu hierarchies (cousin to
GLANCE → CARD → README). *If* a living HCI guest consents to join, compare humane attention vs
calm technology (Weiser memorial).

### 3. Canon Cat vs GitHub

Honest contrast: Raskin wanted to **hide** the file system from casual users. We **expose** it as
stage. The README overlay is the compromise — museum prose without lying about the yaml girders
underneath.

### 4. CS547 — Terry's shelf

would join a tribute episode. Connect to Will's 1996 talk on the same seminar lineage.

### 5. View Source + humane layer
