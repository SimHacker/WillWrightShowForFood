# Humane interface × this repo — two layers, one path

Jef Raskin 🕯️ (1943–2005) argued for the **humane interface**: protect the user's **locus of
attention**, cut arbitrary cognitive load, avoid mode traps. *The Humane Interface* (2000), the
**Canon Cat**, and **Archy** are the anchors. Terry Winograd hosted him at CS547 —
*Prolegomena to future interface design* (2001-10-05) — [`../terry-winograd/media/cs547-ARCHIVE.md`](../terry-winograd/media/cs547-ARCHIVE.md).

## Brutalist listing + humane README

Every folder here has **two faces**:

| GitHub shows first | On a Repo Show you want |
|--------------------|-------------------------|
| **Directory listing** — filenames, yaml, indexes | **`README.md`** — curated links, context, browse order |

Same path; two projections. Listing = **view source** (inspectable yaml girders, verify CI). README =
**human overlay** (museum prose, cross-links). On air: *view source → jump to README → museum.*

## Honest synthesis (not a Canon Cat)

Raskin wanted to **hide** the file system from casual users. This show **is** the repo — RTFR,
verify, yaml girders. We **overlay** navigation instead of pretending the tree isn't there.
